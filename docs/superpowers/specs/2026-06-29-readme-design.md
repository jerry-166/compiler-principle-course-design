# README + 文档体系设计

**日期**：2026-06-29
**状态**：已批准，已实施

## 目标

为编译原理课程设计工程建立清晰的 README 和文档体系，让三个读者（自己、同伴、答辩老师）都能快速理解项目结构和每个文件的阶段归属。

## 决策记录

1. **README 结构**：简洁索引型（方案 A）——项目概述 + 目录树（带阶段标签）+ 交付清单 + 文档索引
2. **阶段标签**：`[PL/0]` `[1.1]` `[1.2]` `[参考]` 等，标注在每个文件旁边
3. **docs 分层**：按实践阶段建子文件夹（`1.1-词法分析/`、`1.2-语法分析/`），跨阶段参考文档保留在根
4. **辅助文档**：`docs/阶段交付总览.md`（按阶段分表）+ `docs/变更日志.md`（按时间倒序）
5. **维护约定**：每完成一个阶段，更新 README 的交付清单状态、阶段交付总览表、变更日志

## 产出物

| 文件 | 用途 |
|------|------|
| `README.md` | 项目总入口 |
| `docs/阶段交付总览.md` | 按阶段分表的文件级归属 |
| `docs/变更日志.md` | 增量变更记录 |
