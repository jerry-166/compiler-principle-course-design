# 实践 2 语义分析 — 设计文档

- 日期：2026-06-29
- 阶段：实践 2（语义分析），对应 Project_2.pdf 第 3 章
- 范围：必做 17 种语义错误 + 选做要求 3.1（函数声明）/ 3.2（嵌套作用域）/ 3.3（结构等价）
- 目标环境：Ubuntu 20.04 + GCC 7.5.0 + Flex 2.6.4 + Bison 3.5.1

## 1. 目标与成功标准

### 1.1 功能目标

为 C-- 语言实现语义分析模块，对**已通过词法/语法分析**的源程序做上下文相关检查与类型检查，按指定格式输出错误。

输入：一个 C-- 源文件路径（命令行参数）。
输出（stdout）：
- 无语义错误：不输出任何内容。
- 有语义错误：每条一行，格式 `Error type [N] at Line [行号]: [说明文字].`，行尾有换行。

### 1.2 成功标准

`Tests/Tests2` 全部 30 个用例（A-1~A-20、B-1~B-2、C-1~C-2、D-1~D-3、E-1~E-3）的程序输出与对应 `.exp` 文件**逐字一致**。

判定标准（PDF 3.3.3）本质是"错误类型 + 行号正确"，但为规避评测脚本做完整字符串匹配的风险，**说明文字完全照搬 Tests2 的 .exp 措辞**。

### 1.3 非目标（YAGNI）

- 不实现指针、面向对象、类型推断、隐式类型转换。
- 不做动态类型检查（C-- 是静态强类型）。
- 不展开数组做结构等价（PDF 要求3.3 明确禁止）。
- 不重新实现词法/语法分析（复用实践 1 的建树产物）。

## 2. 总体架构

### 2.1 编译流水线

```
test.cmm → flex(lexer.l) → bison(parser.y 建语法树) → 语义分析遍历(sem.c) → 错误输出
```

语义分析在**已有的语法树**上做后序遍历，不重新分析词法/语法。

### 2.2 可执行程序：两个独立 main 共享建树代码

| 可执行 | 用途 | 链接 |
|---|---|---|
| `C/bin/parser`（实践1，已有） | 建树 + 打印树 | lex.yy.c + parser.tab.c + tree.c + main_tree.c |
| `C/bin/cmmc`（实践2，新建） | 建树 + 语义分析 | lex.yy.c + parser.tab.c + tree.c + symtab.c + type.c + sem.c + main_sem.c |

**关键改动：从 parser.y 移出 main。** parser.y 现在含 `main`（建树后打印树）。为让两个程序各自带自己的 main 且共享 `yyparse`，把 main 从 parser.y 删除，改为：
- `C/src/syntax/main_tree.c`：实践1的 main（建树→`printTree`→`flush_output`）
- `C/src/sem/main_sem.c`：实践2的 main（建树→`sem_analyze(root)`→`flush_output`）

parser.y 保留所有建树 action、`yyerror`、全局 `root`、`has_error`。

### 2.3 选做 3.1 对 parser.y 的改动

要求 3.1 需要支持函数声明（`int func(int a);`），C-- 原始文法无此产生式。新增：

```
ExtDef : Specifier FunDec SEMI   /* 函数声明，新增产生式 */
```

该产生式建出语法树节点（FunDec 子树），由 sem.c 识别为"声明"。**新增产生式不影响实践1测试**：实践1的输入不含函数声明，走不到这条产生式；原有产生式优先级不变。需重新 `bison` 后确认 `%expect` 冲突数仍可控。

## 3. 数据结构

### 3.1 类型系统（type.h，按 PDF 3.2.4）

```c
typedef struct Type_* Type;
typedef struct FieldList_* FieldList;

struct Type_ {
    enum { BASIC, ARRAY, STRUCTURE } kind;
    union {
        int basic;                       /* 0=int, 1=float */
        struct { Type elem; int size; } array;
        FieldList structure;
    } u;
};

struct FieldList_ {
    char *name;
    Type type;
    FieldList tail;
};
```

- BASIC：`basic` 字段 0/1 区分 int/float。
- ARRAY：链表式多维。`int a[10][3]` → 外层 array(size=10, elem=内层 array(size=3, elem=int))。逐维拆解。
- STRUCTURE：域用 FieldList 链表。
- 类型等价 `type_equal(a,b)`：
  - BASIC：basic 值相等。
  - ARRAY：维数相同且每维元素类型 `type_equal` 递归相等（PDF：基类型+维数相同即匹配，size 不要求相等，如 `int a[10][2]` 与 `int b[5][3]` 同型）。
  - STRUCTURE：**名等价**（默认假设5）比结构体名；**结构等价**（要求3.3）逐域比 name+type，**不展开数组**（PDF 例：数组域只比元素类型与维数，不递归展开元素）。

### 3.2 符号表（symtab.h，函数式栈 + 哈希）

```c
enum { SYM_VAR, SYM_FUNC, SYM_STRUCT };

typedef struct Symbol_* Symbol;
struct Symbol_ {
    char *name;
    int   kind;                /* SYM_VAR / SYM_FUNC / SYM_STRUCT */
    Type  type;                /* 变量/结构体的类型；函数的返回类型 */
    FieldList params;          /* SYM_FUNC 的形参链表（名字+类型） */
    int   declared_only;       /* 要求3.1：1=仅声明未定义 */
    int   depth;               /* 所在作用域深度 */
    Symbol next_hash;          /* 同一哈希桶的链表 */
};
```

- 哈希：pjw 函数（PDF 3.2.3 给出），桶数 0x3fff。冲突用拉链法。
- **作用域栈（函数式）**：进入 `CompSt`（函数体/语句块）压一层新作用域，离开弹一层并删除该层所有符号。栈深度 = 嵌套层数。查表从栈顶往下，返回第一个匹配（内层遮蔽外层）。
- 变量/函数/结构体**共用一张表**，用 `kind` 区分。错误3（变量重名）/4（函数重名）/16（结构体重名）的判断需要跨 kind 查"名字是否已被任何种类占用"。
- 结构体域名不进主符号表（假设7：域不与变量重名、不同结构体域不重名），仅在结构体类型内部用 FieldList 维护；错误15（域重名）在收集域时检查。

## 4. 遍历与检查逻辑

入口 `void sem_analyze(Node *root)`。递归遍历语法树，按节点名 dispatch。

### 4.1 顶层结构处理

| 语法结构 | 动作 |
|---|---|
| `Program → ExtDefList` | 递归处理每个 ExtDef；全部处理完后扫描符号表，对 declared_only 的函数报错误18 |
| `ExtDef → Specifier ExtDecList SEMI` | Specifier→Type；遍历 ExtDecList 逐个建变量类型（含数组维度）填表，重名报错误3/16 |
| `ExtDef → Specifier FunDec SEMI`（3.1新增） | 填函数符号 declared_only=1；与已有声明/定义冲突报错误19；重名报错误4 |
| `ExtDef → Specifier FunDec CompSt` | 填函数符号；冲突报错误19/4；压新作用域，形参填表，遍历 CompSt，弹作用域 |
| `StructSpecifier → STRUCT OptTag LC DefList RC` | 收集 DefList 中域→FieldList（域重名报错误15）；建 STRUCTURE 类型；结构体名查表重名报错误16，填表 |
| `Def → Specifier DecList SEMI` | Specifier→Type；遍历 DecList，每个 Dec 的 VarDec 建变量类型填当前作用域表（重名报错误3） |

### 4.2 语句处理

| 语句 | 检查 |
|---|---|
| `Stmt → RETURN Exp SEMI` | 当前函数返回类型 vs `check_exp(Exp)`，不匹配报错误8 |
| `Stmt → IF LP Exp RP Stmt` / `WHILE LP Exp RP Stmt` | 条件 Exp 须 int（假设2），否则错误7 |
| `Stmt → CompSt` | 压/弹作用域，递归处理内部 DefList/StmtList |

> 返回类型上下文：遍历时维护 `current_func_return_type`，进入函数定义时设置，return 检查时读取。

### 4.3 表达式检查（核心：`Type check_exp(Node *exp)`）

`check_exp` 返回表达式的 Type，语义出错时返回 NULL（NULL 传播短路，避免连锁报错）。

| Exp 形式 | 检查与返回类型 |
|---|---|
| `ID` | 查表；无→错误1（Undefined Variable），返回 NULL；有→返回变量类型 |
| `INT` | 返回 int |
| `FLOAT` | 返回 float |
| `ID LP Args RP` / `ID LP RP` | 查表；无→错误2（Undefined function）；非函数→错误11；是函数→实参逐个比类型/数量，不符→错误9；返回函数返回类型 |
| `Exp ASSIGNOP Exp` | 左侧须左值（见4.4），否则错误6；左右 `type_equal`，否则错误5；返回左侧类型 |
| `Exp AND/OR/RELOP Exp` | 两边须 int（假设2），否则错误7；返回 int |
| `Exp PLUS/MINUS/STAR/DIV Exp` | 两边须 int 或 float 且同型（假设2），否则错误7；返回操作数类型 |
| `MINUS Exp` / `NOT Exp` | NOT 须 int；MINUS 须 int/float；否则错误7 |
| `Exp LB Exp RB` | 左侧须 ARRAY（否则错误10）；下标 Exp 须 int（否则错误12）；返回数组元素类型 |
| `Exp DOT ID` | 左侧须 STRUCTURE（否则错误13）；域须存在（否则错误14）；返回域类型 |
| `LP Exp RP` | 返回内部 Exp 类型 |

### 4.4 左值判定（错误6）

从语法层判断（PDF 3.2.5）：左值 Exp 的产生式只能是以下三种，其余均为右值：
- `ID`
- `Exp LB Exp RB`（数组访问）
- `Exp DOT ID`（结构体域访问）

出现在赋值号左边的其他形式（`INT`、`FLOAT`、`Exp + Exp`、函数调用等）→ 错误6。

### 4.5 错误抑制策略

- `check_exp` 返回 NULL 时，上层不基于 NULL 继续报类型错误（NULL 传播短路）。
- 发现一个错误不退出，继续遍历找其它错误（PDF 3.2.5）。
- 但同一表达式只报一个本质错误，靠短路控制连锁反应。
