- 日期：2026-06-30（设计定稿；按课程时间线 7/2 启动实现）
- 阶段：实践 3（中间代码生成），对应 Project_3.pdf 第 4 章

## 1. 目标与成功标准

### 1.1 功能目标

为 C-- 语言实现中间代码生成器。把**已通过词法/语法/语义分析**的 C-- 源程序翻译为**线性三地址代码**（PDF 4.1.3、4.2.1 的四元式文本形式），输出到文件。

输入：一个 C-- 源文件路径 + 一个输出文件路径（命令行参数）。
输出（写入输出文件）：每行一条中间代码，符合 PDF 表 4.6 的 17 种语句格式。

### 1.2 输入假设（PDF 4.3.1，必做七假设）

1. 无注释、八进制/十六进制整型常数、浮点型常数或变量
2. 无结构体类型变量、无高维数组（高于一维）
3. 函数参数只能是简单变量（结构体/数组不作为参数）
4. 无全局变量、所有变量不重名
5. 函数不返回结构体或数组
6. 函数只定义一次（无函数声明）
7. 输入文件不含任何词法/语法/语义错误（函数必有 return）

选做要求 4.1 放宽假设 2/3：允许结构体变量、结构体作形参。
选做要求 4.2 放宽假设 2/3：允许一维数组作形参、允许高维数组变量。

### 1.3 成功标准

`Tests/Tests3` 全部 16 个用例（A-1~A-5、B-1~B-3、C-1~C-2、E1-1~E1-3、E2-1~E2-3）的程序运行结果（write 序列）与 Tests.pdf 期望**逐字一致**。

**关键差异**：Tests3 没有 `.exp` 期望 IR 文件，期望输出是**程序运行后的 write 序列**（如 A-1 期望 `[120, 325]`）。验证链：我的 IR → irsim 虚拟机执行 → 比对 write 序列。

### 1.4 非目标（YAGNI）

- 不做语义检查（假设 7 保证输入无错误）。3.0 不复用 2.0 的错误检查逻辑。
- 不做代码优化（实践 5 任务）。
- 不实现浮点运算（假设 1）。
- 不实现指针类型（C-- 假设不含指针，IR 中的 `&`/`*` 仅用于数组/结构体寻址）。
- 不实现全局变量（假设 4）。
- 不做回填（PDF 4.2.5 明确：用继承属性传 `label_true`/`label_false`，不使用回填技术）。

## 2. 总体架构

### 2.1 编译流水线

```
test.cmm
  → flex(lexer.l) → bison(parser.y 建语法树)
  → translate 遍历语法树，调用 gen() 追加 IR 节点到全局链表
  → ir.c 把链表逐行打印到输出文件
```

3.0 在**已有的语法树**上做后序遍历（与 2.0 的遍历思路同构），但**副作用是生成 IR 节点而非检查错误**。

### 2.2 可执行程序：三 main 共享建树代码

| 可执行 | 用途 | 链接 |
|---|---|---|
| `C/bin/parser`（实践 1） | 建树 + 打印树 | lex.yy.c + parser.tab.c + tree.c + main_tree.c |
| `C/bin/cmmc`（实践 2） | 建树 + 语义分析 | + symtab.c + type.c + sem.c + main_sem.c |
| `C/bin/cmmc_ir`（实践 3，新建） | 建树 + 生成 IR | + type.c + ir/*.c + main_ir.c |

**关键约束**：3.0 不再需要 symtab.c（2.0 的符号表是为错误检查服务的，3.0 假设 7 无错误，只需轻量映射）。但需要 type.c 的类型计算（数组总大小、结构体字段偏移）。

### 2.3 单可执行 vs 双可执行（选做策略）

2.0 用双可执行是因为 D-1/E-1 期望输出**互斥**（函数声明在必做/选做下行为相反）。3.0 的选做**没有这种互斥**：

- 必做模式下遇到结构体/高维数组，按 PDF 要求输出 `Cannot translate: ...` 提示。
- 选做模式下正常翻译。

两种模式行为一致（都是"能翻就翻，不能翻就提示"），差异只在能否翻译结构体/高维数组。**用条件编译开关 + 单可执行**最简：

```c
/* translate_addr.c 内部 */
#ifdef ENABLE_STRUCT_AND_ARRAY
    /* 翻译结构体、高维数组 */
#else
    /* 检测到这些结构，调用 cannot_translate() */
#endif
```

但为了和 Tests2 验证策略一致（必做 cmmc 测必做行为、cmmc_full 测选做），**仍采用双可执行**：
- `cmmc_ir`（必做）：不开宏，遇到结构体/高维数组输出 Cannot translate
- `cmmc_ir_full`（选做）：开 `ENABLE_STRUCT`、`ENABLE_HIGH_DIM_ARRAY` 宏，正常翻译

理由：双可执行在 2.0 已踩过坑、路径熟；选做功能复杂，隔离到独立可执行便于排查；评测时 A/B/C 跑 cmmc_ir，E1/E2 跑 cmmc_ir_full。

## 3. 数据结构

### 3.1 IR 指令链表（ir.h，按 PDF 表 4.6）

```c
typedef enum {
    IR_LABEL,      /* LABEL x :                  */
    IR_FUNCTION,   /* FUNCTION f :               */
    IR_ASSIGN,     /* x := y                     */
    IR_ADD,        /* x := y + z   (含 -, *, /)  */
    IR_SUB,
    IR_MUL,
    IR_DIV,
    IR_ADDR,       /* x := &y                    */
    IR_LOAD,       /* x := *y                    */
    IR_STORE,      /* *x := y                    */
    IR_GOTO,       /* GOTO x                     */
    IR_IF,         /* IF x [relop] y GOTO z      */
    IR_RETURN,     /* RETURN x                   */
    IR_DEC,        /* DEC x [size]               */
    IR_ARG,        /* ARG x                      */
    IR_CALL,       /* x := CALL f                */
    IR_PARAM,      /* PARAM x                    */
    IR_READ,       /* READ x                     */
    IR_WRITE       /* WRITE x                    */
} IRKind;

typedef struct IR_ *IR;
struct IR_ {
    IRKind kind;
    Operand x, y, z;   /* 操作数（按 kind 用不同字段；不用全） */
    char  *name;       /* IR_FUNCTION/IR_CALL 的函数名；IR_GOTO/IR_LABEL/IR_IF 的标号名 */
    Relop relop;       /* IR_IF 的关系运算符 */
    int    size;       /* IR_DEC 的字节数 */
    IR     next;       /* 全局链表后继 */
};
```

全局链表头 `ir_head`、尾 `ir_tail`。`gen_xxx(...)` 系列函数构造节点并追加到尾部，对应 PDF 的"增量翻译 Gen() 函数"。

### 3.2 Operand（operand.h）

PDF 4.2.4 的 `place` 可以是变量、临时变量或立即数。统一为 Operand：

```c
typedef enum {
    OP_VAR,       /* 源码变量：name = "v_a" 等 */
    OP_TEMP,      /* 临时变量：name = "t1" 等 */
    OP_CONST,     /* 立即数：value，打印为 "#value" */
    OP_LABEL      /* 标号：name = "label1" 等（仅 IR_IF/GOTO/LABEL 用） */
} OperandKind;

typedef struct {
    OperandKind kind;
    char *name;     /* OP_VAR/OP_TEMP/OP_LABEL */
    int   value;    /* OP_CONST */
} Operand;
```

**生成器**（全局计数器）：
- `new_temp()` → 返回 `t1`、`t2`、...（OP_TEMP）
- `new_label()` → 返回 `label1`、`label2`、...（OP_LABEL）
- `new_var(const char *src_name)` → 返回 `v_<src_name>`（OP_VAR），并查轻量符号表绑定类型

**取地址操作数**：PDF 5/6 规则——`ARG`/`CALL` 传数组/结构体参数时用 `&v_a`。Operand 不直接表示 `&v`，而是在 `gen_arg` 时按变量类型决定是否前缀 `&`：`ARG &v_a` 用一个特殊标记（如 `OP_VAR` 加 `by_ref` 标志，或 `OP_ADDR` 种类）。实现选 `OP_ADDR` 种类更清晰：

```c
/* 新增 */
OP_ADDR,   /* &name，表示变量地址 */
```

打印规则：
- `OP_VAR`/`OP_TEMP`/`OP_LABEL` → 直接打印 name
- `OP_CONST` → `#value`
- `OP_ADDR` → `&name`

### 3.3 轻量符号表（translate.c 内部静态）

不复用 2.0 的 symtab（错误检查专用，3.0 用不上）。3.0 只需"变量名 → IR 名 + 类型"映射：

```c
typedef struct {
    char *src_name;   /* 源码变量名 "a" */
    char *ir_name;    /* IR 名 "v_a" */
    Type  type;       /* 类型（数组用得着：总大小、维数；结构体用得着：字段偏移） */
} VarEntry;
```

PJW 哈希（复用 2.0 hash 函数，桶数 0x3fff），扁平表（无作用域栈，假设 4 不重名）。

`lookup_var(src_name)` → 返回 VarEntry*；`insert_var(src_name, type)` → 创建 v_src_name 并入表。

进入函数体时遍历 DefList 填表（含数组类型计算总字节数）。形参也填表（IR 名同样是 v_<name>）。

### 3.4 类型系统复用（type.h/type.c）

完全复用 2.0 的 type.c（已过 30/30 测试）。3.0 额外需要两个工具：

- `type_size(Type t)`：类型的字节宽度。int=4；数组=元素宽度×size 递归；结构体=逐域宽度累加。
- `struct_field_offset(FieldList fields, const char *fname, int *out_offset)`：在结构体域链表中找 fname，输出其相对结构体首地址的偏移，返回域类型。

这两个工具放 translate_addr.c（只有它们用），不污染 type.c 的纯净。

## 4. 翻译逻辑（核心：translate.h/c）

入口 `void translate_program(Node *root)`：遍历 Program → ExtDefList，对每个 ExtDef 调 `translate_extdef`。

### 4.1 顶层结构处理

| 语法结构 | 翻译动作 |
|---|---|
| `Program → ExtDefList` | 逐个 ExtDef 调 `translate_extdef` |
| `ExtDef → Specifier ExtDecList SEMI`（全局变量） | **假设 4 不出现**；若出现，按 DEC 处理（但假设 4 禁止全局变量，可不实现） |
| `ExtDef → Specifier FunDec CompSt`（函数定义） | `translate_fundef`：输出 `FUNCTION <name> :`；形参逐个 `PARAM v_<name>` + 填表；遍历 CompSt |
| `ExtDef → Specifier SEMI`（结构体定义） | 不产生 IR（仅类型定义） |

### 4.2 函数定义 translate_fundef

```
1. 输出 IR_FUNCTION name
2. 收集形参列表（FunDec → VarList → ParamDec → Specifier VarDec）
3. 每个形参：输出 IR_PARAM v_<name>，insert_var(name, type)
4. 进入 CompSt：
   a. 遍历 DefList：每个局部变量 insert_var；若是数组/结构体，输出 IR_DEC v_<name> <size>
   b. 遍历 StmtList：每个 Stmt 调 translate_stmt
```

**DEC 大小计算**：`int buf[6]` → type_size = 6×4 = 24 → `DEC v_buf 24`。结构体按字段宽度累加。PDF 4.3.1 规则 9：DEC 大小为 4 的倍数（int 数组和结构体天然满足）。

### 4.3 语句翻译 translate_stmt（PDF 表 4.2）

| 语句产生式 | 翻译 |
|---|---|
| `Stmt → Exp SEMI` | `translate_exp(Exp, NULL)`（place=NULL 表示丢弃结果） |
| `Stmt → CompSt` | `translate_compst`（处理内层 DefList + StmtList；作用域扁平，不压栈） |
| `Stmt → RETURN SEMI` | `RETURN #0`（C-- 文法实际是 RETURN Exp SEMI，见下行） |
| `Stmt → RETURN Exp SEMI` | `t = new_temp(); translate_exp(Exp, t); RETURN t` |
| `Stmt → IF LP Exp RP Stmt` | L1=new_label,L2=new_label; `translate_cond(Exp,L1,L2); LABEL L1; translate_stmt(Stmt); LABEL L2` |
| `Stmt → IF LP Exp RP Stmt1 ELSE Stmt2` | L1,L2,L3=new_label; `cond(Exp,L1,L2); L1; stmt1; GOTO L3; L2; stmt2; L3` |
| `Stmt → WHILE LP Exp RP Stmt` | 按 PDF 表 4.2 严格顺序：L1,L2,L3=new_label; `LABEL L1; cond(Exp,L2,L3); LABEL L2; translate_stmt(Stmt); GOTO L1; LABEL L3`（L1=循环头回跳点，L2=条件为真入口，L3=循环退出） |

### 4.4 表达式翻译 translate_exp（PDF 表 4.1，translate_addr.c）

`void translate_exp(Node *exp, Operand place)`。按 Exp 产生式分派：

| Exp 形式 | 翻译（往 place 放值） |
|---|---|
| `INT` | `place := #<ival>`（place 为 NULL 时跳过） |
| `ID`（变量） | `place := v_<id>`（查表得 IR 名） |
| `ID LP RP`（无参函数调用） | 特判 read：`READ place`；否则 `place := CALL <name>` |
| `ID LP Args RP`（带参函数调用） | 特判 write：翻译 Args[0] 到 t，`WRITE t`，`place := #0`；否则见 translate_args |
| `Exp1 ASSIGNOP Exp2` | 左值分三类（见 4.5）；普通 ID：`t1=new_temp; translate_exp(Exp2, t1); v_<id> := t1; if(place) place := v_<id>` |
| `Exp1 PLUS Exp2` | `t1=new_temp; t2=new_temp; translate_exp(Exp1,t1); translate_exp(Exp2,t2); place := t1 + t2` |
| `Exp1 MINUS/STAR/DIV Exp2` | 同上换 op |
| `MINUS Exp1`（一元负） | `t1=new_temp; translate_exp(Exp1,t1); place := #0 - t1` |
| `Exp1 RELOP Exp2` | 数值化：`place := #0; translate_cond(Exp, L_true, L_false); L_true: place := #1; L_false:` |
| `Exp1 AND/OR Exp2` | 同 RELOP，走 translate_cond |
| `NOT Exp1` | 同 RELOP，走 translate_cond |
| `Exp1 LB Exp RB`（数组下标） | 见 translate_addr |
| `Exp1 DOT ID`（结构体域） | 见 translate_addr（选做 4.1） |
| `LP Exp RP` | `translate_exp(Exp1, place)`（括号不产生代码） |

### 4.5 左值赋值的三种情况

赋值 `Exp1 ASSIGNOP Exp2` 中 Exp1 的产生式决定如何写入：

| Exp1 形式 | 翻译 |
|---|---|
| `ID` | 普通变量：`v_<id> := t1` |
| `Exp_arr LB Exp_idx RB` | 数组元素：算地址 `addr = &v_arr + offset`，`*addr := t1` |
| `Exp_base DOT ID_field` | 结构体域：算地址 `addr = &v_base + field_offset`，`*addr := t1` |

提取公共函数 `Operand translate_lvalue_addr(Node *lvalue)` 返回地址操作数（普通变量返回 OP_VAR；数组/结构体元素返回计算后的 OP_TEMP，内容是地址）。赋值时按返回类型分派 `:= ` 或 `*:=`。

### 4.6 数组/结构体寻址 translate_addr（PDF 4.2.7）

**一维数组 `a[i]`**（必做）：
```
t_idx = new_temp; translate_exp(i, t_idx);
t_off = new_temp; t_off := t_idx * #4;
t_addr = new_temp; t_addr := &v_a + t_off;
/* 读：t_val := *t_addr；写：*t_addr := t_val */
```

**多维数组 `a[i][j][k]`**（选做 4.2）：
按 PDF 4.2.7 公式 `ADDR(a[i][j][k]) = ADDR(a) + Σ i_t × SIZE(a[t])`。需要类型链上每维的"剩余宽度"。

算法：从最外层维度向内逐维累加。维护 `(addr, type)` 二元组：
```
addr = &v_a; type = a 的类型（ARRAY）
while (type->kind == ARRAY) {
    t_idx = new_temp; translate_exp(当前下标 Exp, t_idx);
    t_off = new_temp; t_off := t_idx * #type_size(type->u.array.elem);
    addr = new_temp; addr := addr + t_off;
    type = type->u.array.elem;
}
/* addr 现在是该元素的地址 */
```

`type_size` 递归计算内层数组的总宽度（如 `int[2][3]` 的内层 `int[3]` 宽度 = 12）。

**结构体域 `s.f`**（选做 4.1）：
```
field_offset = struct_field_offset(s 的类型, "f", &field_type);
addr = new_temp; addr := &v_s + #field_offset;
/* 读：t := *addr；写：*addr := t */
```

**嵌套结构体 `s.u.p`**（选做 4.1）：递归——先算 `s.u` 的地址，再加 `p` 的偏移。
**结构体数组 `arr[i].f`**：先算 `arr[i]` 地址，再加 `f` 偏移。
**数组结构体 `s.arr[i]`**：先算 `s.arr` 地址（s 首地址 + arr 域偏移），再按数组寻址。

通用模式：`translate_addr(node)` 返回 `(addr, type)`，递归处理嵌套。

### 4.7 函数调用与参数 translate_args（PDF 表 4.4/4.5）

```
translate_args(Args, arg_list):
    t = new_temp; translate_exp(Args->Exp, t);
    把 t 插入 arg_list 头部;  /* 逆序：第一个翻译的实参最后 ARG */
    if (Args 还有 COMMA Args1) translate_args(Args1, arg_list);

调用点（ID LP Args RP）：
    arg_list = NULL; translate_args(Args, arg_list);
    特判 write：仅 1 个参数，直接 WRITE arg_list[0]，place := #0
    否则：for (a in arg_list) ARG a;   /* 已是逆序，按列表顺序 ARG */
         place := CALL <name>
```

**read/write 特判**：read 无参数，直接 `READ place`；write 一个参数，`WRITE t`，place 固定填 `#0`（PDF 表 4.4）。

**数组/结构体作参数**（选做 4.1/4.2）：`ARG` 传地址而非值。在 `ARG` 生成时检查实参类型——若是 ARRAY/STRUCTURE，把操作数转为 `OP_ADDR`（`ARG &v_a`）。PDF 4.3.1 规则 10 明确。

### 4.8 条件表达式 translate_cond（PDF 表 4.3）

`void translate_cond(Node *exp, Operand label_true, Operand label_false)`。

| Exp 形式 | 翻译 |
|---|---|
| `Exp1 RELOP Exp2` | `t1=new_temp; t2=new_temp; translate_exp(Exp1,t1); translate_exp(Exp2,t2); IF t1 <relop> t2 GOTO label_true; GOTO label_false` |
| `NOT Exp1` | `translate_cond(Exp1, label_false, label_true)`（真假互换） |
| `Exp1 AND Exp2` | `L=new_label; translate_cond(Exp1, L, label_false); LABEL L; translate_cond(Exp2, label_true, label_false)` |
| `Exp1 OR Exp2` | `L=new_label; translate_cond(Exp1, label_true, L); LABEL L; translate_cond(Exp2, label_true, label_false)` |
| 其他（数值表达式当条件） | `t=new_temp; translate_exp(Exp, t); IF t != #0 GOTO label_true; GOTO label_false` |

## 5. 输出格式

### 5.1 文件输出

`main_ir.c` 接收两个命令行参数：`./cmmc_ir <input.cmm> <output.ir>`。

```c
int main(int argc, char **argv) {
    /* 1. yyin = fopen(argv[1]); yyparse(); 建树 */
    /* 2. translate_program(root);   全局 ir_head 链表填好 */
    /* 3. FILE *out = (argc >= 3) ? fopen(argv[2], "w") : stdout; */
    /* 4. ir_print(out);   遍历 ir_head，每条指令按格式打印一行 */
    /* 5. return 0; */
}
```

### 5.2 IR 语句打印格式（PDF 表 4.6 严格）

| IRKind | 打印模板 |
|---|---|
| IR_LABEL | `LABEL %s :` |
| IR_FUNCTION | `FUNCTION %s :` |
| IR_ASSIGN | `%s := %s` |
| IR_ADD/SUB/MUL/DIV | `%s := %s [+|-|*|/] %s` |
| IR_ADDR | `%s := &%s` |
| IR_LOAD | `%s := *%s` |
| IR_STORE | `*%s := %s` |
| IR_GOTO | `GOTO %s` |
| IR_IF | `IF %s [%s] %s GOTO %s`（relop 文字 `<`/`<=`/`>`/`>=`/`==`/`!=`） |
| IR_RETURN | `RETURN %s` |
| IR_DEC | `DEC %s %d` |
| IR_ARG | `ARG %s` |
| IR_CALL | `%s := CALL %s` |
| IR_PARAM | `PARAM %s` |
| IR_READ | `READ %s` |
| IR_WRITE | `WRITE %s` |

**操作数打印**：变量/临时/标号直接打 name；常量打 `#value`；地址打 `&name`。

**空格规则**（PDF 4.3.1）：赋值号前后、运算符前后、关键字与变量之间都要有空格或制表符。`IF x relop y GOTO z` 中各 token 之间都有空格。`LABEL x :` 中 `LABEL` 与 `x`、`x` 与 `:` 都被空格隔开。

### 5.3 Cannot translate 提示（选做开关）

未开 `ENABLE_STRUCT` 时遇到结构体变量/参数、未开 `ENABLE_HIGH_DIM_ARRAY` 时遇到高维数组/数组参数：

**统一提示语**（结构体和高维数组共用 PDF 原话）：

```
Cannot translate: Code contains variables of multi-dimensional array type or parameters of array type.
```

**输出流**：stderr。理由：(1) 不污染 `.ir` 输出文件，虚拟机读文件时不会把提示当成 IR 代码报错；(2) 符合 Unix 错误信息走 stderr 的惯例；(3) PDF 措辞"提示信息"是给人看的，不是给虚拟机执行的。

**何时检测**：在产生任何 IR 输出**前**做一遍 prepass 扫描所有变量声明；若发现不支持的结构（结构体类型变量、高维数组变量、结构体/数组作形参），输出提示后**跳过** translate_program 主流程，不产生任何 IR 输出（避免输出半截 IR）。

## 6. 测试策略

### 6.1 验证机制（核心创新点）

Tests3 期望输出是**程序运行结果**（write 序列），不是 IR 文本。验证链：
```
我的 cmmc_ir 生成 out.ir → irsim 虚拟机执行 out.ir（带 stdin 输入）→ 收集所有 WRITE 输出
                            → 与 Tests.pdf 期望的 write 序列对比
```

**irsim 部署**（实现期第一任务）：
- `irsim.pyc` 是 Python 3.8 + PyQt5 程序，本机 Python 3.14 不兼容。
- 方案：`uv python install 3.8` + 创建虚拟环境跑 `python irsim.pyc`。
- 优先验证 irsim 是否支持 CLI 模式（接受 `.ir` 文件 + stdin → stdout 输出 write 序列）。
- 若仅 GUI：fallback 到自写 IR 解释器（C 实现，仅支持必做 17 种语句）。

**irsim 的输入约定**（Tests.pdf 记号）：每个 READ 一行、每个 WRITE 一行。Tests.pdf 写在同一行仅为节省版面，实际评测各占一行。

### 6.2 测试三件套（按 memory 协议）

- **测试指南** `docs/3.0-中间代码生成/测试指南.md`：编译命令、单跑命令、irsim 运行方法、批量测脚本用法。
- **批量脚本** `操作手册及脚本/run_tests_ir.sh`（英文目录）：遍历 Tests/Tests3，跑 cmmc_ir 生成 .ir，喂给 irsim，对比 write 序列，汇总通过率。
- **测试记录** `docs/3.0-中间代码生成/测试记录.md`：每个用例的输入代码 + 生成的 IR + irsim 运行输出 + 期望 + 通过/失败。

### 6.3 关键测试场景覆盖

- **A 组（5 个，简单）**：
  - A-1：while 循环 + 算术累加 + write（无 read，无数组，无函数）→ 测基础表达式+循环
  - A-2：多次 read + 多 if + 算术组合 → 测 read/算术/if
  - A-3：一维数组初始化 + 下标 read → 测数组寻址（必做核心）
  - A-4：二重 while + 嵌套条件 → 测嵌套控制流标号
  - A-5：递归函数 fib → 测函数调用 + ARG/CALL/PARAM
- **B 组（3 个，中等）**：
  - B-1：数组求最大值下标 + 辅助函数 → 综合数组+函数
  - B-2：二重循环枚举 + 多参数函数 → 测多参数 ARG 逆序
  - B-3：函数 + 数组变换 + 乘积 → 综合测试
- **C 组（2 个，经典）**：
  - C-1：辗转相除 GCD → 测 while + 整除/取余（用 `/` `*` 实现 `%`）
  - C-2：digitSum → 测辅助函数 + 循环取位
- **E1 组（选做 4.1 结构体，3 个）**：
  - E1-1：简单结构体 Rect → 测结构体域寻址
  - E1-2：结构体作形参 → 测 ARG &v + *addr 读写
  - E1-3：嵌套结构体 → 测多层域偏移
- **E2 组（选做 4.2 高维数组+数组参数，3 个）**：
  - E2-1：二维数组 g[2][3] → 测多维寻址
  - E2-2：一维数组作形参 + 多次调用 → 测 ARG &arr + 函数内 *addr
  - E2-3：三维数组 + 一维数组形参 → 综合高维+参数

### 6.4 必做两个 PDF 样例（关键回归点）

Project_3.pdf 给了两个必做样例的期望 IR（sgn、fact）。虽然变量名不强求一致，但**翻译模式应一致**。实现期把这两个样例输入我的 cmmc_ir，人工对比翻译结构（不是逐字，是模式对应）：
- sgn（PDF 4.3.6 样例 1）：if-else 嵌套、read、write、RETURN
- fact（PDF 4.3.6 样例 2）：递归函数、PARAM、ARG、CALL、IF/GOTO/LABEL

## 9. 风险与对策

| 风险 | 对策 |
|---|---|
| irsim 是 GUI 程序不支持 CLI | 实现期第一任务验证；fallback 自写极简 IR 解释器（C，仅必做 17 语句） |

## 10. 与 2.0 的代码关系

### 10.1 复用

- `C/src/syntax/`（lexer.l、parser.y、tree.c、node.h）—— 完全复用，不改
- `C/src/sem/type.h`、`type.c` —— 完全复用（数组维数、结构体字段、类型构造函数）

## 11. 实现任务拆分（预览，writing-plans 阶段细化）
