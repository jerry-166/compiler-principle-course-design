# 实践 2 语义分析 — 测试指南

> 配套可执行：`C/bin/cmmc`（必做）+ `C/bin/cmmc_full`（选做全开）
> 测试集：`Tests/Tests2/inputs/*.cmm`（30 个）+ `Tests/Tests2/expects/*.exp`
> 批量脚本：`操作手册及脚本/run_tests_sem.sh`

## 1. 环境要求

| 组件 | 版本 |
|---|---|
| GCC | 7.5.0（目标 Ubuntu 20.04 自带；高版本也能编但行为以低版本为准） |
| Flex | 2.6.4（生成 `lex.yy.c`；仓库已提交生成产物，可不重跑） |
| Bison | 3.5.1（生成 `parser.tab.c/h`；仓库已提交） |
| 编译选项 | `-std=c99 -Wall`（C99 严格模式，禁 POSIX 扩展如 strdup） |

> WSL2 Ubuntu 24.04 (GCC 13.3 / Bison 3.8) 也能正常编译运行；只要源码不引入低版本不支持的高版本特性即可。

## 2. 编译两个可执行

**必做 cmmc**（不开任何选做宏）：

```bash
gcc -std=c99 -Wall -o C/bin/cmmc \
    C/src/syntax/parser.tab.c C/src/syntax/tree.c \
    C/src/lex/lex.yy.c \
    C/src/sem/type.c C/src/sem/symtab.c C/src/sem/sem.c C/src/sem/main_sem.c \
    -I C/src/syntax -I C/src/sem -lm
```

**选做 cmmc_full**（三宏全开：函数声明 + 嵌套作用域 + 结构等价）：

```bash
gcc -std=c99 -Wall -o C/bin/cmmc_full \
    C/src/syntax/parser.tab.c C/src/syntax/tree.c \
    C/src/lex/lex.yy.c \
    C/src/sem/type.c C/src/sem/symtab.c C/src/sem/sem.c C/src/sem/main_sem.c \
    -DENABLE_FUNC_DECL -DENABLE_SCOPE -DENABLE_STRUCT_EQ \
    -I C/src/syntax -I C/src/sem -lm
```

**重新生成 bison/flex**（一般不需要，仓库已提交生成产物）：

```bash
flex  -o C/src/lex/lex.yy.c           C/src/lex/lexer.l
bison -d -o C/src/syntax/parser.tab.c C/src/syntax/parser.y
```

> 重新生成后**必须重新编译 parser 可执行**并跑 Tests1 回归（确保未影响实践 1）：
> `bash tmp_run_tests1.sh` 应输出 `Tests1: 20 passed, 0 failed`。

## 3. 单跑一个用例

```bash
# 必做：A-1 是 "Undefined Variable"
./C/bin/cmmc Tests/Tests2/inputs/A-1.cmm
# 输出（与 A-1.exp 逐字一致）：
# Error type 1 at Line 15: Undefined Variable.

# 选做：E-1 是函数声明错误
./C/bin/cmmc_full Tests/Tests2/inputs/E-1.cmm
# 输出：
# Error type 18 at Line 4: Undefined function.
# Error type 19 at Line 21: Inconsistent declaration.

# 比对期望
diff <(./C/bin/cmmc Tests/Tests2/inputs/A-1.cmm) Tests/Tests2/expects/A-1.exp
```

注意 `.exp` 文件可能是 **CRLF** 行尾，程序输出是 **LF**。`diff` 直接比可能有"虚假差异"，需先归一化：

```bash
got=$(./C/bin/cmmc Tests/Tests2/inputs/A-1.cmm | tr -d '\r')
exp=$(cat Tests/Tests2/expects/A-1.exp | tr -d '\r')
[ "$got" = "$exp" ] && echo PASS || echo FAIL
```

## 4. 批量测试

一键脚本（编译 + 全量测试 + 汇总）：

```bash
bash 操作手册及脚本/run_tests_sem.sh
```

脚本逻辑：

1. 编译 cmmc 和 cmmc_full（失败立即报错退出）。
2. 分组跑：
   - `cmmc @ A`（20 个）
   - `cmmc @ B`（2 个）
   - `cmmc @ C`（2 个）
   - `cmmc @ D`（3 个）
   - `cmmc_full @ E`（3 个）
3. 逐个 diff .exp（行尾归一化），输出 PASS/FAIL + 首行 diff。
4. 汇总：30/30 为目标。

预期输出：

```
=== cmmc @ A 系列 ===
  PASS A-1
  PASS A-2
  ...
  -> A: 20/20 passed
=== cmmc @ B 系列 ===
  -> B: 2/2 passed
=== cmmc @ C 系列 ===
  -> C: 2/2 passed
=== cmmc @ D 系列 ===
  -> D: 3/3 passed
=== cmmc_full @ E 系列 ===
  -> E: 3/3 passed
================================
总计：30 passed, 0 failed
```

## 5. 测试用例分组

| 组 | 用例数 | 描述 | 用哪个可执行 |
|---|---|---|---|
| A | 20 | 每种语义错误类型单独触发（A-1~A-20 对应错误1~错误17，部分错误有多个用例） | cmmc |
| B | 2 | 多错误综合用例（B-1 错误7+6+13+14、B-2 错误7+10+8） | cmmc |
| C | 2 | 无错误程序，验证零输出（C-1 全局变量+结构体、C-2 递归函数） | cmmc |
| D | 3 | 必做行为：D-1 函数声明报 type B、D-2 嵌套块重定义报错3、D-3 名等价报错5 | cmmc |
| E | 3 | 选做行为：E-1 函数声明错误18/19、E-2 嵌套作用域错误9/10、E-3 结构等价错误8/5 | cmmc_full |

**判定标准**：实际输出与 .exp **逐字一致**（行尾归一化后）。一行不多、一行不少、每个字符都对应。

## 6. 为什么 cmmc 测 D 而 cmmc_full 测 E

| 用例 | cmmc（不开宏）行为 | cmmc_full（三宏全开）行为 |
|---|---|---|
| D-1 函数声明 | 报 type B `Syntax error.`（不开 ENABLE_FUNC_DECL，产生式报错） | 合法（开 ENABLE_FUNC_DECL），继续做语义分析 |
| D-2 嵌套块重定义 | 报错误3 `Redefined variable or structure conflict.`（不开 ENABLE_SCOPE，嵌套视为同作用域） | 合法遮蔽（开 ENABLE_SCOPE），不报错 |
| D-3 名等价 | 报错误5 `Type mismatched for assignment.`（默认名等价，v1 ≠ v2） | 不报错（开 ENABLE_STRUCT_EQ，结构等价下 v1 ≡ v2） |
| E-1 函数声明 | 报 type B（必做模式无函数声明产生式） | 报错误18+19（合法声明，做声明检查） |
| E-2 嵌套作用域 | 嵌套块内重定义报错3（干扰测试） | 合法遮蔽，按选做语义报错9+10 |
| E-3 结构等价 | 报错误8+5（名等价下默认行为） | 报错误8+5（结构等价下也按域差异报） |

> **E-3 在两种模式下输出相同**（巧合：E-3 的错误场景在两种等价判据下都触发）。

## 7. 常见问题排查

| 现象 | 可能原因 | 排查 |
|---|---|---|
| 全部用例 FAIL（无输出+段错误） | lexer.l 用了 strdup（隐式声明返回 int，64 位截断指针） | 已修复：lexer.l 用 dup_yytext 替代；若重新 flex 后又出现，检查 lexer.l 顶部 |
| C-2 报 `Redefined Variable` 行5 | visit_compst 按索引取 DefList 但被空产生式 NULL 错位 | 按节点名识别（visit_compst 现已这么写） |
| A-19 多报错误5 | PLUS/MINUS 类型不符时返回非 NULL，上层连锁报错误5 | 错误7 后短路返回 NULL |
| E-1 错误18 在错误19 之后 | 错误18 最后扫描输出 | 两阶段 prepass 先收集"会被定义的函数名"，主遍历声明时立即按行号报 |
| 行尾 diff 差异（CRLF vs LF） | .exp 是 CRLF，输出是 LF | 比较前 `tr -d '\r'` 归一化 |

## 8. 与实践 1 共存的注意事项

实践 1 的 `parser` 可执行（建树+打印树）和实践 2 的 `cmmc` / `cmmc_full` **共享** `parser.tab.c / tree.c / lex.yy.c`。修改这些共享文件后：

1. 重新跑 `parser` 的 Tests1 回归：`bash tmp_run_tests1.sh` → 应 20/20
2. 重新跑 `cmmc` / `cmmc_full` 的 Tests2 回归：`bash 操作手册及脚本/run_tests_sem.sh` → 应 30/30

任何一个回归失败都说明共享代码改坏了，需要修复后再继续。
