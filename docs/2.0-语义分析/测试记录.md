## 3. 验证要点

### 3.1 错误18/19 输出顺序（E-1）

E-1 期望先报错误18（行4，foo 声明但未定义），后报错误19（行21，bar 二次声明与定义冲突）。本项目用**两阶段扫描**：

- prepass 先扫一遍 ExtDefList 收集所有"会被定义"的函数名到 `defined_func_names`；
- 主遍历遇到函数声明时，若名字不在 `defined_func_names` 立即按声明行号报错误18。

E-1 实际输出（行号顺序正确）：

```
Error type 18 at Line 4: Undefined function.
Error type 19 at Line 21: Inconsistent declaration.
```

### 3.2 错误3 两种措辞区分（A-3 vs D-2）

| 用例 | 场景 | nest_depth | 措辞 |
|---|---|---|---|
| A-3 | 函数体顶层同作用域重定义 | 1 | `Redefined Variable.` |
| D-2 | 嵌套 `{}` 块内重定义外层变量 | 2 | `Redefined variable or structure conflict.` |

`nest_depth` 是独立的语法嵌套计数器（与符号表作用域栈解耦），进函数体 +1，进 Stmt→CompSt 嵌套块再 +1。

### 3.3 名等价 vs 结构等价（D-3 vs E-3）

| 用例 | 模式 | 行为 |
|---|---|---|
| D-3 | cmmc（名等价） | `v1 = v2` 报错误5（不同结构体定义产生不同 Type 指针） |
| E-3 | cmmc_full（结构等价） | 仍报错误8+5×2（域顺序不同/元素类型不同的结构体不等价） |

D-3 行29 `v1 = v2`（v1 是 struct T1{int,float}，v2 是 struct T2{int,float}）：
- 名等价：v1.type ≠ v2.type（不同指针） → 错误5
- 结构等价：v1 和 v2 域类型相同（都 int,float） → 等价 → 不报错

但 D-3 期望报错误5 → 名等价正确。E-3 行27 `s1 = s2`（s1 是 S1{int,float}，s2 是 S2{float,int}）：
- 结构等价：逐域比 type，第一个域 int vs float → 不等价 → 错误5

E-3 期望错误5 → 结构等价的逐域比较正确。

### 3.4 类型短路避免连锁错误（A-19）

A-19 行14 `g1.mat[0][0] = vec + g1.nr;`，其中 `vec` 是数组：

- `vec + g1.nr` 触发错误7（数组参与算术），check_exp **返回 NULL**（短路）
- 上层 ASSIGNOP 看到 rhs=NULL，**不基于 NULL 继续报错误5**

期望只报错误7 一条，实际也是只报错误7 一条。✓

## 5. 已知设计取舍

| 取舍 | 说明 |
|---|---|
| cmmc_full 不过 D 系列 | 设计如此：开 ENABLE_FUNC_DECL 后函数声明合法（D-1 应报 type B 但变合法）；开 ENABLE_SCOPE 后嵌套重定义合法（D-2 应报错3 但变合法遮蔽）；开 ENABLE_STRUCT_EQ 后同结构等价（D-3 应报错5 但变等价）。cmmc_full 是选做行为，**不应用于测 D 系列** |
| cmmc 不过 E 系列 | 设计如此：必做模式无函数声明产生式（E-1 应报错18/19 但走 type B）；无作用域栈（E-2 嵌套合法变非法）；名等价（E-3 部分场景报告不同）。cmmc 是必做行为，**不应用于测 E 系列** |
| E-3 在两种模式下输出相同 | 巧合：E-3 的错误场景（return 类型不匹配 + 同结构不同域顺序赋值不匹配）在名等价和结构等价下都触发，所以 cmmc 和 cmmc_full 都过 E-3 |
