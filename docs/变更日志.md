- **新增** `C/src/sem/type.h` / `type.c` — 类型系统（BASIC/ARRAY/STRUCTURE，名等价 vs 结构等价由 `ENABLE_STRUCT_EQ` 切换）
- **新增** `C/src/sem/symtab.h` / `symtab.c` — 符号表（pjw 哈希 + 函数式作用域栈，`ENABLE_SCOPE` 控制是否真正压栈）
- **新增** `C/src/sem/sem.h` / `sem.c` — 语义分析主体（visit_extdef/visit_stmt/check_exp，错误 1–19 全实现）
- **新增** `C/src/sem/main_sem.c` — cmmc / cmmc_full 入口（建树 → sem_analyze → flush）
- **新增** `C/bin/cmmc` — 必做可执行（无宏），过 Tests2 A/B/C/D 27 用例
- **新增** `C/bin/cmmc_full` — 选做可执行（三宏全开），过 Tests2 A/B/C/E
- **新增** `docs/2.0-语义分析/语义分析.md` — 语义分析原理 + 实现文档
- **新增** `docs/2.0-语义分析/测试指南.md` — 用例分类 + 编译运行步骤
- **新增** `docs/2.0-语义分析/测试记录.md` — 30 个用例逐个记录（与 .exp 逐字一致）
- **新增** `操作手册及脚本/run_tests_sem.sh` — 批量测试脚本（必做 + 选做一体）

### 测试结果

Tests2 共 30 个用例，cmmc + cmmc_full 覆盖全部用例，与 `.exp` **逐字一致**：

| 分组 | 用例数 | 可执行 | 通过率 |
|---|---|---|---|
| A（单错误）| 20 | cmmc | 100% |
| B（多错误综合）| 2 | cmmc | 100% |
| C（无错误）| 2 | cmmc | 100% |
| D（必做行为）| 3 | cmmc | 100% |
| E（选做行为）| 3 | cmmc_full | 100% |

### 关键设计

- 两阶段扫描保证错误 18（函数声明未定义）按行号顺序先于错误 19 输出
- 独立的 `nest_depth` 计数器（与符号表作用域栈解耦）区分错误 3 的两种措辞
- 表达式类型不符时返回 NULL 短路传播，避免基于错误类型连锁报错
- 共享实践 1 的 `parser.tab.c / tree.c / lex.yy.c`，Tests1 回归 20/20 PASS
