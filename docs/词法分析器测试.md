# 词法分析器测试文档

> 配套代码：`C/src/lex/lexer.l`
> 适用阶段：C-- 编译器第 1 阶段（词法分析，实践 1.1）
> 测试集位置：`Tests/`（三套：Tests1、Tests2、Tests3）

---

## 1. 测试集概览

`Tests/` 目录下有三套测试，对应编译器的不同阶段。**词法分析阶段只用 Tests1**，Tests2 和 Tests3 是后面阶段（语义分析、中间代码生成）用的。

| 测试集 | 期望输出内容 | 适用阶段 | 备注 |
|--------|-------------|---------|------|
| **Tests1** | 语法树 / 词法错误(Error type A) / 语法错误(Error type B) | **词法 + 语法**（实践 1.1 + 1.2） | **本次测试用这套** |
| Tests2 | 语义错误(Error type 1~14) | 语义分析（实践 2） | 后续阶段 |
| Tests3 | 仅有 inputs，无 expects | 中间代码生成（实践 3） | 后续阶段 |

### Tests1 测试用例分类

Tests1 下的测试用例按字母前缀分类，每个类别侧重不同的语言特性：

| 类别 | 用例 | 测试重点 |
|------|------|---------|
| **A 系列** | A-1 ~ A-10 | **合法程序**，各语言特性（函数调用、数组、结构体、浮点、十六进制、全局变量、递归、科学计数等）。期望输出是完整的语法树。词法分析阶段只需确认 Token 全部正确识别、无 Error type A。 |
| **B 系列** | B-1, B-2 | 包含注释（单行 `//` 和多行 `/* */`），测试注释能否正确丢弃。期望输出是语法树（注释不应出现在树中）。 |
| **C 系列** | C-1, C-2 | 包含**词法错误**（未定义字符）。期望输出是 `Error type A`。**词法分析阶段的核心测试**。 |
| **D 系列** | D-1 ~ D-3 | 包含**语法/语义错误**（各种 Error type B）。这些错误由后续阶段检测，词法分析只需确认没有漏报/错报 Error type A。 |
| **E1 / E2 / E3 系列** | E1-1~2, E2-1~2, E3-1~2 | 更大的合法程序（结构体、斐波那契、栈）。期望输出是语法树。测试复杂程序下的 Token 识别稳定性。 |

### Tests2 测试用例分类（参考，本次不测）

Tests2 有 A 系列（A-1~A-20，语义错误 type 1~14）、B 系列（语法错误）、C 系列、D 系列、E 系列（错误修正后的代码）。**全部是语义分析阶段的测试**。

## 2. 词法分析器测试步骤

### 2.1 编译词法分析器

```bash
# 第一步：用 Flex 把 .l 文件生成 C 代码
flex -o C/src/lex/lex.yy.c C/src/lex/lexer.l

# 第二步：用 gcc 把生成的 C 代码编译成可执行文件
gcc C/src/lex/lex.yy.c -o C/bin/scanner
```

编译成功后会在 `C/bin/` 下生成 `scanner` 可执行文件。编译过程中不应有 warning 或 error。

### 2.2 运行测试

对任意 `.cmm` 输入文件执行词法分析：

```bash
# 测试单个文件
./C/bin/scanner Tests/Tests1/inputs/A-1.cmm

# 也可以测试项目自带的样例
./C/bin/scanner C/src/lex/test_tokens.cmm
```

### 2.3 词法分析器的输出格式

词法分析器逐行输出 Token，格式如下：

| Token 类型 | 输出格式 | 示例 |
|-----------|---------|------|
| 关键字 | `TYPE` / `IF` / `WHILE` / `RETURN` / `ELSE` / `STRUCT` | `TYPE` |
| 标识符 | `ID: 名称` | `ID: main` |
| 整数 | `INT: 十进制值` | `INT: 123`、`INT: 255`（0xFF） |
| 浮点数 | `FLOAT: 原始字符串` | `FLOAT: 3.14`、`FLOAT: 1.05e-4` |
| 运算符 | `PLUS` / `MINUS` / `STAR` / `DIV` / `ASSIGNOP` / `RELOP` / `AND` / `OR` / `NOT` | `PLUS` |
| 符号 | `SEMI` / `COMMA` / `DOT` / `LP` / `RP` / `LB` / `RB` / `LC` / `RC` | `SEMI` |
| 词法错误 | `Error type A at Line X: ...` | `Error type A at Line 3: Undefined character "$".` |

注释和空白字符不产生任何输出（被丢弃）。

## 3. 重点测试用例说明

### 3.1 合法程序测试（A 系列）

**目的**：验证所有 Token 类型的识别是否正确、无漏报/错报。

推荐测试顺序：

| 文件 | 内容特点 | 重点关注 |
|------|---------|---------|
| A-1 | 函数调用、十六进制(`0x0000`/`0xFFFF`)、八进制(`0755`) | INT 的十六进制/八进制转十进制输出正确 |
| A-2 | 全局数组、while 嵌套、单行+多行注释 | 注释被正确丢弃，不出现在输出中 |
| A-3 | 结构体定义、成员访问(`.`)、while | STRUCT、DOT 识别正确 |
| A-4 | 简单函数、浮点赋值 | FLOAT 识别正确 |
| A-5 | 浮点数(科学计数法 `9.1e-2`/`2.99e8`) | 选做 2.2 指数浮点识别 |
| A-6 ~ A-10 | 更多合法程序，递增复杂度 | 综合测试 |

**验证方法**：对合法程序（A 系列），词法分析器的输出中**不应出现任何 Error type A**。每行输出都应该是合法的 Token。

### 3.2 词法错误测试（C 系列）—— 最重要

**目的**：验证词法分析器能正确检测并报告非法字符。

**C-1（`Tests/Tests1/inputs/C-1.cmm`）**：

输入第 3 行包含未定义字符 `$`：

```c
int step = base $ 3;    // $ 是 C-- 未定义的字符
```

期望输出：

```
Error type A at Line 3: Mysterious characters "$".
```

**C-2（`Tests/Tests1/inputs/C-2.cmm`）**：

包含多个词法/语法/语义错误（`$` 符号、尾逗号、匿名结构体等）。词法分析阶段关注的是**第 3 行的 `$`**：

```
Error type A at Line 3: Undefined character "$".
```

### 3.3 注释测试（B 系列）

**目的**：验证 `//` 单行注释和 `/* */` 多行注释能被正确丢弃。

**B-1（`Tests/Tests1/inputs/B-1.cmm`）**：

文件包含两种注释：

```c
/* Matrix utility: supports init and trace */
/* All values default to zero on init */
...
mat[i][j] = val; /* assign element */
...
return mat[1][1];    // return center value
```

验证：输出中不应出现注释内容的 Token（如 `Matrix`、`utility`、`return`（来自注释）不应作为 ID 或关键字出现）。注意区分注释中的文字和实际代码中的 `return`。

### 3.4 自带测试样例

`C/src/lex/test_tokens.cmm` 覆盖了大部分 Token 类型：

```c
int main() {
  int i = 1;
  float a = 3.14;
  a = .5;           // . 和 5 会被拆成 DOT + INT:5
  if (i >= 1 && j <= 0) { ... }
  while (i != 0) { ... }
  return 0;
}
```

这个文件适合作为**快速验证**用例，跑一遍确认基本功能正常。

## 4. 测试检查清单

逐项检查，全部通过即表示词法分析器实现正确：

- [ ] 编译成功，无 warning / error
- [ ] A 系列所有文件（A-1 ~ A-10）输出无 Error type A
- [ ] C-1 正确报错 `Error type A at Line 3`（`$` 字符）
- [ ] C-2 正确报错 `Error type A at Line 3`（`$` 字符）
- [ ] B-1 注释内容不出现在输出中（`//` 和 `/* */` 都正确丢弃）
- [ ] 十六进制整数转换正确（如 `0xFF` → `INT: 255`）
- [ ] 八进制整数转换正确（如 `0755` → `INT: 493`）
- [ ] 指数浮点识别正确（如 `9.1e-2` → `FLOAT: 9.1e-2`）
- [ ] 所有关键字不被误识别为 ID（`int` → `TYPE` 而非 `ID: int`）
- [ ] 多字符运算符正确识别（`>=` → `RELOP`，不被拆成 `>` + `=`）
