> 测试集位置：`Tests/Tests1/`（inputs/ + expects/）
> 主文档：[`语法分析.md`](语法分析.md) ｜ 逐用例记录：[`测试记录.md`](测试记录.md)

## 1. 测试集概览

`Tests/` 下有三套测试，**语法分析阶段只用 Tests1**（和词法分析 1.1 同一套，但本阶段关注的输出不同）。

| 测试集 | 期望输出内容 | 适用阶段 |
|--------|-------------|---------|
| **Tests1** | 语法树 / 词法错误(Error type A) / 语法错误(Error type B) | **词法 + 语法**（1.1 + 1.2） |
| Tests2 | 语义错误(Error type 1~14) | 语义分析（实践 2） |
| Tests3 | 仅有 inputs，无 expects | 中间代码生成（实践 3） |

### 1.1 Tests1 用例分类（语法分析视角）
本阶段 parser 对每个输入会输出三种结果之一：**完整语法树**（合法程序）、**Error type B**（语法错误，只打错误信息不建树）、**Error type A**（词法错误，由 lexer 直接打印）。

| 类别 | 用例 | parser 输出类型 | 测试重点 |
|------|------|----------------|---------|
| **合法建树** | C-1, C-2, D-1~D-3, E1-1, E2-1, E3-1, E3-2 | 完整语法树 | 树结构、行号、INT/FLOAT 值格式 |
| **B 类语法错误** | A-3, A-4, A-6~A-10, B-1, B-2 | Error type B（多条） | 精确错误信息 + 不打印树 |
| **A 类词法错误** | A-1, A-2, A-5, E1-2, E2-2 | Error type A | lexer 层报错（沿用 1.1） |

> **注意**：Tests1 的字母前缀（A/B/C/D/E）**不代表**"A 类就是词法错误"。实际分类以 parser 输出为准——比如 A 系列里 A-1/A-2/A-5 是词法错误，A-3/A-4/A-6~A-10 是语法错误。这是因为同一前缀的用例可能混合多种错误。

## 2. 编译与运行

### 2.1 编译语法分析器

```bash
# 1. Bison 把 .y 生成 C 代码（parser.tab.c + parser.tab.h）
bison -d -o C/src/syntax/parser.tab.c C/src/syntax/parser.y

# 2. Flex 把 .l 生成 C 代码（lex.yy.c）
flex -o C/src/lex/lex.yy.c C/src/lex/lexer.l

# 3. gcc 把三个 C 文件编译成 parser
gcc -o C/bin/parser \
    C/src/syntax/parser.tab.c \
    C/src/syntax/tree.c \
    C/src/lex/lex.yy.c \
    -I C/src/syntax -lm
```

编译成功后在 `C/bin/` 下生成 `parser`。编译过程不应有 warning/error。

> **版本要求**（协作约束）：Bison ≥ 3.5.1、Flex ≥ 2.6.4、GCC ≥ 7.5。代码已避免使用 3.8/13 才有的特性。详见 [`语法分析.md`](语法分析.md) 第 5 节踩坑 6。

### 2.2 运行单个用例

```bash
# 合法程序 → 打印语法树
./C/bin/parser Tests/Tests1/inputs/C-1.cmm

# 语法错误 → 打印错误信息（不打印树）
./C/bin/parser Tests/Tests1/inputs/B-1.cmm
```

> `parser` 是 Linux ELF 二进制，需在 WSL Ubuntu 中运行。

### 2.3 批量验证（推荐）

项目自带两个验证脚本，自动遍历 Tests1 所有用例并与 expects 做 diff：

```bash
bash C/scripts/verify.sh          # 容许 CRLF 差异（内容比对，推荐日常用）
bash C/scripts/verify_strict.sh   # 严格逐字节比对（不容许任何差异）
```

输出示例：

```
PASS A-1
KNOWN A-10 (test-data: CRLF/extra-newline)
PASS A-2
...
---
PASS=19  REAL_FAIL=0  KNOWN(test-data)=1  MISSING(exp)=3
```

各状态含义见 [`测试记录.md`](测试记录.md)。

## 3. parser 的输出格式

### 3.1 合法程序：缩进语法树

每层缩进 2 空格。节点格式见主文档 [`语法分析.md`](语法分析.md) 2.2 节：

| 节点类型 | 格式 | 示例 |
|---------|------|------|
| 非终结符 | `Name (lineno)` | `Program (1)`、`Stmt (4)` |
| 终结符-有值 | `TOKEN: value` | `INT: 65280`、`FLOAT: 3.140000`、`ID: main`、`TYPE: int` |
| 终结符-无值 | `TOKEN` | `SEMI`、`LP`、`IF`、`RETURN` |

- **INT 按十进制打印**：`0xFF00`→`65280`、`0644`→`420`（进制转换在词法层做）。
- **FLOAT 按 `%f`（6 位小数）打印**：`3.14e0`→`3.140000`、`6.02e23`→`602000017271895229464576.000000`（float32 语义，详见主文档踩坑 2）。
- 树末尾带一个 `\n`（与 Tests1 树 .exp 一致）。

### 3.2 错误程序：错误信息

- **Error type A**（词法错误，lexer 打印）：`Error type A at Line X: <具体描述>`。
- **Error type B**（语法错误，parser 打印）：`Error type B at Line X: <具体描述>`。
- 错误信息末尾**不带** `\n`（与 Tests1 错误 .exp 一致；A-10 是例外，详见测试记录）。
- **遇到任何 type B 错误，整棵树都不打印**——只打印错误信息。

完整的 18 条精确错误信息清单见 [`测试记录.md`](测试记录.md) "错误信息清单"。

## 4. 重点用例说明

### 4.1 合法建树用例（C/D/E 系列）

**目的**：验证树结构、行号传播、INT/FLOAT 值格式完全正确。

| 用例 | 内容特点 | 重点关注 |
|------|---------|---------|
| C-1 | 简单函数 + if 嵌套 | 树的层次结构、行号取首子节点 |
| D-1 | 浮点运算 | FLOAT 按 `%f` 打印 |
| D-2 | 科学计数法 `6.02e23` | **float32 精度**（最易错，见主文档踩坑 2） |
| D-3 | 数组、结构体成员访问 | VarDec 嵌套、DOT 节点 |
| E1-1, E2-1, E3-1 | 较大程序（结构体/斐波那契/栈） | 复杂程序下建树稳定性 |

**验证方法**：输出应是完整语法树，与 expects/*.exp 逐字节一致（`verify_strict.sh` 应 PASS）。

### 4.2 B 类语法错误用例（核心测试）

**目的**：验证每类语法错误都能被**精确捕获**（错误信息逐字匹配 Tests1），且不打印树。

| 用例 | 包含的错误 |
|------|-----------|
| A-3 | `return;` 缺返回值 |
| A-4 | 全局变量初始化 |
| A-6 | 数组维度是浮点 |
| A-7 | 函数调用实参尾逗号 |
| A-8 | 复合赋值 `+=` |
| A-9 | 语句块中语句后声明 |
| A-10 | `a.` 后缺成员名 |
| B-1 | 数组维度浮点 + 形参尾逗号 + 匿名结构体 + 数组初始化 + while 空体（**5 种错误混合**） |
| B-2 | 多余 `]` + 语句后声明 |

每个错误的精确信息和实现方式见 [`测试记录.md`](测试记录.md) "错误信息清单"。**B-1/B-2 内容完全正确，但因 .exp 是 CRLF 编码在 strict 模式下判 FAIL**——这是测试数据问题，不是代码问题。

### 4.3 A 类词法错误用例

A-1（`$` 字符）、A-2（`2x` 数字开头）、A-5（`3.14.15` 多小数点）、E1-2（非法十六进制/八进制）、E2-2（指数浮点缺尾数）。这些由 lexer 直接打印（沿用 1.1 实现），parser 不参与。详见 [`../1.1-词法分析/词法分析.md`](../1.1-词法分析/词法分析.md)。

## 5. 测试检查清单

逐项检查，全部通过即表示语法分析器实现正确：

- [ ] Bison + Flex + gcc 三步编译成功，无 warning / error
- [ ] C/D/E 系列合法程序输出完整语法树，树结构与 expects 一致
- [ ] 非终结符节点行号 = 第一个子节点行号（如 `Program (1)`、`ExtDef (7)`）
- [ ] INT 按十进制打印（`0xFF00`→`65280`、`0644`→`420`）
- [ ] FLOAT 按 `%f` 6 位小数打印（`6.02e23`→`602000017271895229464576.000000`）
- [ ] 无值 Token（SEMI/LP/IF）只打印名字，不带行号
- [ ] B 类错误用例：打印精确错误信息，**不打印任何树**
- [ ] B-1 的 5 种混合错误全部捕获、信息逐字匹配
- [ ] 形参尾逗号 `int f(int a,)` 报 `Trailing comma in function parameters`
- [ ] 匿名结构体 `struct {...};` 报 `Anonymous struct declaration`
- [ ] `verify.sh` 内容比对：19/20 PASS（B-1/B-2 内容匹配）
- [ ] `verify_strict.sh` 严格比对：17/23 PASS（3 个为测试数据问题，详见测试记录）
